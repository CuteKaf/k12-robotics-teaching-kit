# 轻量化项目原则

目标不是先做“最完整的机器人系统”，而是先做“最适合教学、最容易跑通、最容易展示成果的版本”。

## 一、轻量化的核心原则

### 1. 先做教学闭环，不先做工程全闭环
优先保证学生能：
- 看见画面
- 跑通命令
- 得到结果
- 理解因果关系

而不是一开始就把所有硬件、通信、标定、抓取都接满。

### 2. 一节课只验证一个核心能力
例如：
- 这节课只验证相机取流
- 这节课只验证 YOLO 2D 检测
- 这节课只验证 XYZ 输出
- 这节课只验证机械臂到达固定 pose

### 3. 先弱化真实抓取，保留动作逻辑
对教学来说，`hover` 和 `fake grasp` 的价值已经很高。
真正抓取可以作为进阶模块。

### 4. 先统一硬件与环境，不追求兼容所有方案
为了教学稳定：
- 固定 Ubuntu 版本
- 固定相机型号
- 固定机械臂方案
- 固定一套命令和目录结构

### 5. 优先保留“可见成果”模块
最值得保留的是：
- 相机画面
- 目标检测框
- XYZ 输出
- 机械臂到位
- 悬停
- 假抓取流程

## 二、建议优先砍掉或延后的部分
- 复杂总控骨架
- 过早引入完整 Web 控制台
- 多种末端执行器兼容
- 多种通信协议并行教学
- 一开始就追求真实 pick-and-place
- 过多第三方 SDK 的混合依赖

## 三、轻量化成功的标准
一个适合教学的轻量版项目，应该满足：
- 2 小时内能完成基础环境验证
- 1 节课内能看到相机+检测效果
- 2~3 节课内能跑到 XYZ / pose 概念
- 4~6 节课内能看到机械臂动作反馈
- 学生和家长都能看见清晰成果
