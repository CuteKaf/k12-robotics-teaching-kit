# 视频 04：项目目录怎么看

## 这条视频的目标
让学生第一次进入项目时，不会只看到一堆文件名，而是能大概知道：
- 哪些目录是干什么的
- 自己当前最应该先关注哪几个部分
- 不需要一开始把所有内容都看懂

## 建议时长
5~7 分钟

## 开场
这一条视频，我们来解决一个很现实的问题：

**第一次打开一个机器人项目时，目录到底应该怎么看？**

很多同学第一次看到项目仓库，最容易产生的感觉就是：
“文件好多，我根本不知道从哪里开始。”

这个反应非常正常。
因为工程项目本来就不是给人第一眼就全部看懂的，
而是需要先建立一个结构感。

## 正文第一段：看项目目录不是逐个文件死抠
先记住一个原则：
第一次看项目目录，不是要把每个文件都搞懂。

你要先做的是：
**知道这个项目的大块分区分别负责什么。**

也就是说，
先看地图，
不要一上来就钻进每一条小路。

## 正文第二段：最常见的几个核心区块
在机器人课程项目里，通常你会看到几类最核心的目录。

### 第一类：环境相关
比如：
- `environment/`
- 安装脚本
- setup 脚本

这部分的作用通常是：
帮助你把项目运行环境准备好。

### 第二类：视觉相关
比如：
- `vision/`
- 相机脚本
- 检测脚本

这部分的作用是：
让系统先“看见”。

### 第三类：标定相关
比如：
- `calibration/`

这部分负责的是：
把相机看到的位置，转换成机器人能理解的位置。

### 第四类：控制相关
比如：
- `control/`

这部分负责的是：
让机械臂真正动起来。

### 第五类：配置相关
比如：
- `config/`

这部分一般保存：
- 参数
- 位姿
- 工作空间约束
- 标定结果路径

## 正文第三段：学生第一次最该看什么
对于你现在这个阶段，
第一次看项目，不要急着去看所有控制脚本和高级功能。

更合适的顺序是：

1. 先看 README
2. 再看环境初始化相关内容
3. 再看自检脚本
4. 再看视觉入口脚本
5. 最后才逐渐进入标定和控制

这个顺序非常重要。
因为它对应的是系统搭建的真实顺序，
不是目录树的排列顺序。

## 正文第四段：为什么不能一开始就看最复杂的部分
很多同学会忍不住先去看：
- 总控脚本
- 抓取脚本
- 最大最复杂的脚本

但这样往往会更容易晕。

因为复杂脚本通常依赖很多前置条件：
- 环境已经配好
- 相机已经通了
- 模型已经能跑
- 标定已经完成
- 机械臂通信已经正常

如果前置条件都还没建立，
你提前去看这些脚本，只会觉得项目非常复杂。

## 结尾
所以第一次看项目目录，不是要“全部看懂”，
而是要先建立这样一种感觉：

- 环境在哪
- 视觉在哪
- 标定在哪
- 控制在哪
- 配置在哪

等这几个大块先建立起来，
后面每一步都会清楚很多。
