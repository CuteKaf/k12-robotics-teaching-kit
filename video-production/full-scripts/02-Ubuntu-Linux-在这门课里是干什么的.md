# 视频 02：Ubuntu / Linux 在这门课里是干什么的

## 这条视频的目标
让学生理解：
- 为什么机器人项目经常使用 Ubuntu / Linux
- 它在这门课里不是“额外增加难度”，而是工程基础
- 学生不需要一开始就什么都懂，但要知道它承担什么作用

## 建议时长
4~6 分钟

## 开场
这一条视频，我们来讲一个很多同学一开始都会问的问题：

**为什么这门课要用 Ubuntu，或者说 Linux？**

很多同学平时更熟悉的是 Windows，
所以第一次看到 Ubuntu，可能会觉得它很陌生，甚至觉得它是不是故意把学习难度变高了。

其实不是。
在机器人项目里，用 Ubuntu 往往是因为它更适合做这类开发。

## 正文第一段：Ubuntu 在这里到底是什么
你可以先把 Ubuntu 理解成一套操作系统。
就像很多同学平时使用 Windows 一样，Ubuntu 也是一种“电脑系统”。

只不过在机器人、AI、嵌入式和服务器开发里，Ubuntu 非常常见。
原因不是因为它看起来更“高级”，而是因为：
- 开发环境更统一
- 很多工具天然支持它
- 对硬件驱动和开发库更友好
- 在工程项目里更稳定

## 正文第二段：这门课里你会拿 Ubuntu 来做什么
在这门课里，Ubuntu 不是用来“研究系统本身”的，
而是用来承载整个机器人项目。

也就是说，很多后面要做的事情，都要在这个环境里完成，比如：
- 创建和管理 Python 环境
- 安装项目依赖
- 运行相机脚本
- 运行目标检测脚本
- 跑标定脚本
- 控制机械臂
- 查看日志和排错

你可以把它理解成：
**这是我们整套机器人系统的工作台。**

## 正文第三段：学生要学到什么程度
这里也要特别说明一下：
我们不是要把每个同学都训练成 Linux 专家。

在课程的前期，你需要掌握的是：
- 怎么打开终端
- 怎么进入目录
- 怎么运行命令
- 怎么查看输出
- 怎么判断成功或失败

也就是说，
你不需要一开始就理解 Ubuntu 的全部原理，
但你需要会在这个环境里工作。

## 正文第四段：为什么这不是额外负担，而是重要能力
很多时候，学生会觉得：
“我明明是来学机器人，为什么还要碰系统和命令行？”

但实际上，这正是机器人项目的一部分。
因为现实里的机器人系统不是只点点鼠标就能完成的，
而是依赖一整套开发环境。

所以学会在 Ubuntu 里做这些操作，并不是绕路，
而是在学习机器人系统真正的工作方式。

## 结尾
所以如果你用一句话理解 Ubuntu 在这门课里的作用，
它不是一个“额外的难点”，
而是：

**承载整个机器人项目运行与开发的基础环境。**
