# 线下机器人课程教学包 v1

## 核心原则

这套课不是“卖视频课”，而是“线下主课 + 视频化教辅系统”。

- **线下负责高价值环节**：装配、调试、答疑、理解工程逻辑
- **视频负责标准化重复内容**：预习、标准操作、复盘
- **任务单负责课堂执行**：每节课都有明确成功标准

---

## 一、每节课的四种材料

### 1. 教师讲义
用于老师/助教备课：
- 本节目标
- 演示顺序
- 时间分配
- 易错点
- 需要提前准备的设备

### 2. 学生任务单
用于学生执行：
- 今天要完成什么
- 关键命令
- 成功标准
- 课后复盘内容

### 3. 辅助视频
分为三类：
- 课前预习
- 课堂标准操作回放
- 课后复盘

### 4. 课堂素材包
包括：
- 项目仓库
- 脚本清单
- 配置文件
- 标定板 PDF
- 参考截图/示意图

---

## 二、视频在整套课里的定位

### 最适合视频化的内容
- Ubuntu / Conda 基础
- 项目目录介绍
- `check_setup.py` 使用
- RealSense 取流检查
- YOLO 检测结果怎么看
- 标定原理简介
- CAN 激活流程
- fake grasp 的标准启动方式
- 常见报错说明

### 更适合线下完成的内容
- 机械结构装配
- 电机/控制板连接
- 网络与硬件连接
- 标定带做
- 动作安全
- 现场调试
- 项目拓展与答疑

---

## 三、建议的线下授课节奏

### 阶段 1：先有反馈
目标：让学生尽快“看见结果”
- 项目总览
- Ubuntu / 环境搭建
- check setup
- RealSense 取流
- YOLO 检测

### 阶段 2：进入工程逻辑
目标：理解为什么不能只靠识别就抓
- 深度与 XYZ
- 坐标系
- 标定原理
- pose 与安全约束

### 阶段 3：进入动作闭环
目标：让系统真正动起来
- can0 激活
- move_flange_pose
- hover_detected_target
- fake grasp

### 阶段 4：成果化与扩展
目标：从项目走向展示/竞赛/论文
- Web 控制台
- 分类扩展
- 项目展示
- 论文结构

---

## 四、建议的课堂结构（单次 90~120 分钟）

### 结构模板
1. 开场回顾（10 分钟）
2. 本节目标说明（5 分钟）
3. 关键概念讲解（15~20 分钟）
4. 老师演示（15 分钟）
5. 学生实操（30~45 分钟）
6. 排错与答疑（15 分钟）
7. 结果展示与小结（10 分钟）
8. 布置课后视频/任务（5 分钟）

---

## 五、对老师的建议
- 不要一开始追求整套闭环
- 每节课只压一个明确成功点
- 课堂上优先保证学生“跑通”
- 视频要短、小、准、可复用
- 统一术语、命令、目录和命名方式

---

## 六、课程成果设计

### 学生可见成果
- 打印并组装机械臂结构件
- 跑通 Ubuntu / Python / Conda 环境
- 获取相机 RGB-D 数据
- 使用 YOLO 检测目标
- 输出目标 XYZ
- 完成标定
- 实现悬停与 fake grasp

### 对外可展示成果
- 项目演示视频
- 课堂照片/装配过程
- 阶段性报告
- 比赛/论文/成果转化材料
